# Incident Response & Security Breach Simulation

## Objective
Simulate a security incident and perform incident response using system logs.

## Tools Used
- Linux Authentication Logs
- Terminal
- SSH

## Incident Simulated
Repeated failed SSH login attempts (brute force simulation).

## Steps Performed
1. Simulated failed SSH logins
2. Analyzed /var/log/auth.log
3. Identified suspicious activity
4. Contained the incident
5. Performed root cause analysis
6. Restored services
7. Documented timeline and report

## Repository Contents
- Incident report
- Incident timeline
- Commands used
- Log samples

## Learning Outcome
This task provided practical experience in:
- Log analysis
- Incident detection
- Incident containment
- Documentation
